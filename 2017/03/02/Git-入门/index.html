Git 入门

什么是Git

  • git: Git是分布式版本控制系统。Git没有中央服务器，每个人的电脑就是一个完整的版本库。工作的时候不需要联网，因为版本都是在自己的电脑上。

  • svn: Svn是集中式版本控制系统。版本库是集中放在中央服务器。工作时首先要从中央服务器那里得到最新的版本，工作完需要把自己的改动推送到中央服务器。集中式版本控制系统是必须联网才能工作，在局域网中，带宽足够大，但在互联网下，网速慢，就不方便了。

  • git和svn的比较

    • git比svn快。
    • git是分布式的， svn是集中式的。这个特性使得git可以离线开发。
    • git把内容按元数据方式存储，而SVN是按文件
    • git的分支比svn的分支方便，安全，快速。

一些概念

  • 工作区：就是你在电脑上看到的目录，比如目录下testgit里的文件(.git隐藏目录版本库除外)。或者以后需要再新建的目录文件等等都属于工作区范畴。

  • 版本库(Repository)：工作区有一个隐藏目录.git,这个不属于工作区，这是版本库。其中版本库里面存了很多东西，其中最重要的就是stage(暂存区)，还有Git为我们自动创建了第一个分支master,以及指向master的一个指针HEAD。

  • 远成仓库协作: ssh是一种网络协议， 用于加密登录。Git服务器采用SSH公钥来进行授权，安装ssh Linux和Mac标配， windows上的Git Bash也带了。

*.sln // 忽略 .sln类型文件
Temp/ // 忽略 Temp路径下文件

git config –global user.name “My Name”
git config –global user.email myEmail@example.com

// 查看文件在缓存与当前工作目录的状态:
git status

// 显示已写入缓存与已修改但尚未写入缓存的改动的区别
git diff

// 查看已缓存的改动
git diff –cached

// 查看已缓存的与未缓存的所有改动
git diff HEAD

// 显示摘要而非整个 diff
git diff –stat

// 添加文件到本地暂存区
git add filename.txt

// 添加所有文件到本地暂存区
git add -A || git add .

// 将暂存区提交到本地HEAD
git commit -m “say something”

// 自动将在提交前将已记录、修改的文件放入缓存区
git commit -am “say something”

// 取消缓存已缓存的内容
// 执行 git reset HEAD 以取消之前 git add 添加，但不希望包含在下一提交快照中的缓存。
git reset HEAD –

// git rm 将文件从缓存区移除
// 默认情况下, 会将文件从缓存区和你的硬盘中（工作目录）删除
git rm

// 从缓存区删除，但在工作目录中留该文件
git rm –cached

稳定版本的代码不会被破坏
不同的功能可以由不同开发者同时开发。
开发者可以专注于自己的分支，不用担心被其他人破坏了环境
在不确定之前，同一个特性可以拥有几个版本，便于比较

// 列出你在本地的分支。你所在的分支的行首会有个星号作标记。 如果你开启了彩色模式，当前分支会用绿色显示
git branch

// 查看所有分支信息
git branch -v

// 新建分支 test
git branch

// 切换到分支 test
git checkout

// 新建并自动切换分支：
git checkout -b feature_x

// 删除分支b
git branch -d name_b

// 强制删除分支
git branch -D branchname

// 合并分支：两个分支 name_a，name_b 把分支b并入分支a
git checkout name_a
git merge name_b

// 将分支推送到远端仓库
git push origin

// 打标签:-a 选项意为“创建一个带注解的标签”
git tag -a v1.0

// 查看标签记录
git tag

// 切换标签
git checkout v1.0

// 列出远端别名
git remote

// 列出远端别名, 以及别名的实际链接地址
git remote -v

// 设置remote
git remote set-url origin

// 为你的项目添加一个新的远端仓库
git remote add
git remote add origin https://github.com/tutorialzine/awesome-project.git

// 删除现存的某个别名
git remote rm

// 从远端仓库提取数据并尝试合并到当前分支
git pull origin master

// 从远端仓库下载新分支与数据
// git pull = git fetch + git merge github/master
git fetch

// 提交修改到远端仓库， 其中master是远端仓库分支名， 可替换成任意分支。
git push origin master

// 显示一个分支中提交的更改记录
git log

// id 很长，但是你并不需要复制整个字符串，前一小部分就够了。
// b10cc1238e355c02a044ef9f9860811ff605c9b4
git show b10cc1238e35

// git diff [commit-from]..[commit-to]
git diff 09bd8cc..ba25c0ff

// 查看某人的提交
git log –author==david

// 一个压缩后的每一条提交记录只占一行的输出
git log –pretty=oneline

// 帮助
git log –help

// git checkout 版本号 文件全路径
git checkout 09bd8cc1 hello.txt

// HEAD是最新提交的别名
git revert HEAD
git revert b10cc123

// 用 HEAD 中的最新内容替换掉你的工作目录中的文件。已添加到暂存区的改动以及新文件都不会受到影响。
git checkout –

// 丢弃你在本地的所有改动与提交
git fetch origin
git reset –hard origin/master

// git删除本地所有未提交的更改，包括修改的、新增的、删除的，还有一些编译生成的临时文件。回到上一版本的干净状态.
// 如果有修改加入暂存区
git reset –hard
git clean -xdf

// 如果没有加入暂存区
git checkout .
git clean -xdf

// 内建的图形化 git：
gitk

// 彩色的 git 输出：
git config color.ui true

// 显示历史记录时，每个提交的信息只显示一行：
git config format.pretty oneline
